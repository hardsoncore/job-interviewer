
<h3>Computed (Вычисляемые сигналы)</h3>

<p>
  <span class="accent">Computed</span> — это сигналы, которые выводят свое значение на основе других сигналов.
  Они ленивые (пересчитываются только при чтении) и мемоизированные (хранят результат до изменения зависимостей).
</p>

<h3>Effect (Эффекты)</h3>

<p>
  <span class="accent">Effect</span> — это функции, которые автоматически запускаются при изменении сигналов, прочитанных внутри них.
  Они полезны для логирования, работы с DOM или синхронизации с <code>LocalStorage</code>.
</p>

<h3>Когда использовать Effect, а когда это антипаттерн?</h3>

<p>
  <strong>Безопасно использовать Effect:</strong> Когда тебе нужно выполнить побочное действие, которое не влияет на рендеринг UI.
  Например, логирование изменений, синхронизация данных с <code>LocalStorage</code>, отправка аналитики.
</p>

<p>
  <strong>Антипаттерн:</strong> Использовать Effect для управления состоянием или рендерингом UI.
  Если внутри эффекта ты меняешь сигналы, от которых зависят шаблоны, ты рискуешь создать бесконечный цикл обновлений.
  Это может привести к серьезным проблемам с производительностью и даже к краху приложения.
</p>
