<p>
  <code>Object.defineProperty()</code> — это статический метод, который позволяет напрямую определить новое свойство на объекте или изменить существующее, а затем возвращает этот объект.
</p>

<p>
  Этот метод полезен, когда вам нужно создать свойство с определенными характеристиками, такими как неизменяемость, не перечисляемость или не конфигурируемость. Он также позволяет создавать геттеры и сеттеры для управления доступом к свойству.
  <br>
  Что касается практического применения, <code>Object.defineProperty()</code> часто используется в библиотеках и фреймворках для создания API с определенными правилами доступа к данным, а также для оптимизации производительности путем контроля над тем, как свойства взаимодействуют с механизмами JavaScript.
</p>

<h4>Синтаксис</h4>

<code class="code">
  Object.defineProperty(obj, prop, descriptor);
</code>

<p>
  <code>obj</code> — объект, на котором будет определено свойство.
</p>
<p>
  <code>prop</code> — имя свойства, которое будет определено или изменено.
</p>
<p>
  <code>descriptor</code> — объект, который описывает конфигурацию свойства. Он может содержать следующие ключи:
</p>

<p>Дескрипторы бывают двух типов: <strong>дескрипторы данных</strong> и <strong>дескрипторы доступа</strong>. В зависимости от типа дескриптора, набор допустимых ключей будет различаться.</p>

<p>Общие ключи дескриптора (common descriptor keys) - доступны для обоих типов</p>
<ul>
  <li><code>enumerable</code> — указывает, будет ли свойство перечисляться при переборе объекта. По умолчанию <code>false</code>.</li>
  <li><code>configurable</code> — указывает, может ли свойство быть удалено или изменено. По умолчанию <code>false</code>.</li>
</ul>

<p>Дескрипторы данных (data descriptors) - описывают классическое значение свойства</p>
<ul>
  <li><code>value</code> — значение свойства. По умолчанию <code>undefined</code>.</li>
  <li><code>writable</code> — указывает, может ли значение свойства быть изменено. По умолчанию <code>false</code>.</li>
</ul>

<p>Дескрипторы доступа (accessor descriptors) - описывают свойства с геттерами и сеттерами</p>
<ul>
  <li><code>get</code> — (по умолчанию <code>undefined</code>): Функция, которая неявно вызывается без аргументов при любом обращении к свойству. Возвращаемое значение этой функции становится значением свойства.</li>
  <li><code>set</code> — (по умолчанию <code>undefined</code>): Функция, которая вызывается с одним аргументом (новым значением) при попытке присвоить свойству новое значение.</li>
</ul>

<p class="info info--orange">
  Важное правило: дескриптор не может быть одновременно и тем, и другим. Если указать <code>value</code> и <code>get</code> в одном вызове, JavaScript выбросит ошибку:

  <code class="code">
    Object.defineProperty(obj, 'prop', {
      value: 42,
      get() { return this._prop; }
    });

    // TypeError: Invalid property descriptor. Cannot both specify accessors and a value or writable attribute
  </code>
</p>

<h4>Пример использования</h4>
<code class="code">
  const obj = {};

  // Определяем свойство 'name' с помощью Object.defineProperty
  Object.defineProperty(obj, 'name', {
    value: 'Alice',
    writable: false, // Свойство 'name' нельзя изменить
    enumerable: true,
    configurable: false // Свойство 'name' нельзя удалить или изменить его конфигурацию
  });

  console.log(obj.name); // Alice

  // Пытаемся изменить значение свойства 'name'
  obj.name = 'Bob';
  console.log(obj.name); // Alice

  // Делаем попытку удалить свойство 'name'
  delete obj.name;
  console.log(obj.name); // Alice
</code>

<h4>Зачем это использовать в повседневной практике?</h4>
<p>
  В обычных продуктовых задачах редко приходится использовать <code>Object.defineProperty</code> напрямую. Для геттеров и сеттеров сейчас чаще используют более лаконичный синтаксис классов или литералов объектов (например, <code>get propName() { ... }</code>).
  <br>
  Однако, понимание того, как работает <code>Object.defineProperty</code>, важно для глубокого понимания JavaScript, особенно если ты планируешь работать с фреймворками, которые активно используют эту функцию для создания реактивных свойств или оптимизации производительности.
</p>

<h4>Использование в Vue 2</h4>

<p>
  В Vue <code>Object.defineProperty</code> используется для создания реактивных свойств, отслеживания изменений в данных и автоматического обновления DOM при изменении этих данных.
</p>

<p>
  <code>Object.defineProperty</code> как фундаментальная основа реактивности — это эксклюзивная черта Vue 2. Остальные фреймворки (даже следующие поколения самого Vue) пошли совершенно другими путями.
</p>
