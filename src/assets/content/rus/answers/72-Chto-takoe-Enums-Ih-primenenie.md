<h3>Что такое Enums?</h3>

<p><span class="accent">Enum</span> (сокращение от enumeration — перечисление) — это структура данных, которая позволяет задать строгий набор именованных констант, логически связанных между собой.</p>

<p>Грубо говоря, это словарь разрешенных состояний или типов для конкретной сущности. Вместо того чтобы использовать в коде «магические числа» (<code>1</code>, <code>2</code>, <code>3</code>) или обычные строки (<code>"active"</code>, <code>"deleted"</code>), ты создаешь единый источник истины.</p>

<h3>Зачем их применять (и какую боль они решают)?</h3>

<h4>1. Самодокументируемый код</h4>
<p>Конструкция <code>if (user.role === UserRole.Admin)</code> читается как обычный английский текст. Это в разы понятнее, чем <code>if (user.role === 1)</code>.</p>

<h4>2. Защита от глупых ошибок</h4>
<p>Если ты опечатаешься в слове (например, напишешь <code>UserRole.Adimn</code>), компилятор сообщит об этом еще до запуска кода. С обычными строками ошибка возникла бы только в рантайме.</p>

<h4>3. Умное автодополнение</h4>
<p>IDE мгновенно подскажет все доступные варианты, как только ты напечатаешь имя перечисления и поставишь точку. Не нужно держать в голове, какие там статусы есть на бэкенде.</p>

<h4>4. Централизованный рефакторинг</h4>
<p>Если бэкенд решил поменять значение статуса с <code>in_progress</code> на <code>processing</code>, ты меняешь это значение ровно в одной строке — внутри самого Enum.</p>

<h3>Виды Enums</h3>
<p>В TypeScript есть несколько видов Enums:</p>

<p><strong>Numeric Enums</strong> — по умолчанию, каждый элемент получает числовое значение, начиная с 0. Например:</p>
<code class="code">
  enum UserRole {
    Admin, // 0
    User,  // 1
    Guest  // 2
  }

  // Ты можешь явно задавать значения, если хочешь:
  enum ResponseStatus {
    Pending = 0,
    Success = 200,
    Failed = 404
  }
</code>

<p><strong>String Enums</strong> — каждый элемент явно задается строкой. Это более читаемый и безопасный вариант:</p>
<code class="code">
  enum UserRole {
    Admin = "ADMIN",
    User = "USER",
    Guest = "GUEST"
  }
</code>

<p><strong>Heterogeneous Enums</strong> — смешанные, где элементы могут быть как строками, так и числами. Обычно их не рекомендуют использовать, так как это может запутать.</p>

<p class="info info--orange">Считаются антипаттерном, так как могут привести к неочевидным ошибкам.</p>

<code class="code">
  enum MixedEnum {
    No = 0,
    Yes = "YES"
  }
</code>

<p><strong>Const Enums</strong> — оптимизированные для производительности, они полностью удаляются при компиляции, а их значения вставляются напрямую в код. Это может быть полезно для часто используемых констант, но стоит использовать с осторожностью, так как они не поддерживают некоторые возможности обычных Enums (например, итерацию). Также встречаются проблемы с отладкой, так как в скомпилированном коде не будет явного объекта Enum. В последнее время их использование стало менее популярным.</p>

<code class="code">
  const enum UserRole {
    Admin = "ADMIN",
    User = "USER",
    Guest = "GUEST"
  }
</code>

<h3>Что "под капотом" и как быть в чистом JS?</h3>
<p>Важно помнить, что в нативном JavaScript встроенных Enums не существует.</p>

<h4>Что происходит при компиляции?</h4>

<p>Когда TypeScript компилирует код, он превращает твои Enums в реальные JavaScript-объекты (создавая так называемые IIFE-функции).</p>
<p>Например, Enum <code>UserRole</code> будет скомпилирован в объект, который можно использовать для доступа к его значениям. Это позволяет сохранить преимущества Enums даже в среде, где их нет изначально.</p>

<code class="code">
  // TypeScript Enum
  enum Role {
    Admin = 0,
    User = 1
  }

  // Скомпилированный JavaScript
  "use strict";
  var Role;
  (function (Role) {
    Role[Role["Admin"] = 0] = "Admin";
    Role[Role["User"] = 1] = "User";
  })(Role || (Role = {}));
</code>

<h4>Как быть в чистом JS?</h4>

<p>Если ты работаешь над проектом без TypeScript (например, в классическом Vue 2 на чистом JS), паттерн Enum легко эмулируется через неизменяемый объект:</p>

<code class="code">
  const TaskStatus = Object.freeze({
    TODO: 'TODO',
    IN_PROGRESS: 'IN_PROGRESS',
    DONE: 'DONE'
  });

  // Использование точно такое же: TaskStatus.DONE
</code>
