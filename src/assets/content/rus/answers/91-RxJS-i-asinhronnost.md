<h3>Что такое RxJS?</h3>
<p><span class="accent">RxJS (Reactive Extensions for JavaScript)</span> — это библиотека для реактивного программирования. Она использует паттерн <strong>Observable (Наблюдаемый объект)</strong>, чтобы сделать работу с асинхронным кодом и событиями более предсказуемой и удобной.</p>

<p>Вместо того чтобы думать о данных как о статических значениях или использовать классические промисы (Promises), которые возвращают ровно один результат, RxJS предлагает думать о данных как о потоках (streams), которые разворачиваются во времени.</p>

<h3>Для чего нужен RxJS?</h3>
<p>
  В frontend-разработке мы постоянно сталкиваемся с асинхронностью: клики пользователей, HTTP-запросы, таймеры, веб-сокеты.
  <br>
  RxJS решает несколько критических проблем:
</p>

<h5>1. Работа с множественными значениями</h5>
<p><code>Promise</code> возвращает данные один раз и завершается. <code>Observable</code> может возвращать данные непрерывно (например, координаты мыши при движении или поток сообщений из чата).</p>

<h5>2. Отмена запросов (Cancellation)</h5>
<p>Промис нельзя отменить "из коробки". В RxJS подписку на поток можно легко отменить (отписаться), что автоматически прервет, например, висящий HTTP-запрос (это можно сделать с помощью оператора <code>takeUntil</code> или <code>unsubscribe()</code>). Это критически важно для производительности.</p>

<h5>3. Сложные манипуляции с данными</h5>
<p>RxJS предоставляет огромный набор готовых инструментов (операторов) для фильтрации, объединения, задержки и трансформации потоков до того, как они достигнут конечного потребителя.</p>

<h5>4. Управление состоянием</h5>
<p>С помощью таких сущностей, как <strong>BehaviorSubject</strong>, очень удобно хранить и реактивно обновлять состояние приложения.</p>

<h3>Базовые сущности</h3>
<p>RxJS строится вокруг нескольких базовых сущностей и десятков функций-операторов. Вот самые важные из них:</p>

<p> - <strong>Observable (Наблюдаемый)</strong>: Сам поток данных. Он ничего не делает, пока на него не подпишутся.</p>

<p> - <strong>Observer (Наблюдатель)</strong>: Объект с методами <code>next</code>, <code>error</code> и <code>complete</code>, который слушает Observable.</p>

<p> - <strong>Subscription (Подписка)</strong>: Результат выполнения <code>observable.subscribe()</code>. Используется для отписки и предотвращения утечек памяти.</p>

<p> - <strong>Subject</strong>: Специальный тип Observable, который позволяет отправлять значения множеству подписчиков одновременно (multicasting).</p>

<h3>Популярные операторы</h3>
<p><strong>Операторы</strong> — это чистые функции, которые позволяют манипулировать потоком. Они объединяются в цепочки с помощью метода <code>pipe()</code>.</p>

<h5>1. Создание потоков:</h5>

<p><code>of(1, 2, 3)</code> — создает поток из переданных аргументов.</p>

<p><code>from([1, 2, 3])</code> — превращает массив, промис или итерируемый объект в поток.</p>

<p><code>interval(1000)</code> — генерирует значения каждую секунду.</p>

<h5>2. Трансформация и фильтрация:</h5>

<p><code>map(x => x * 2)</code> — трансформирует каждое значение потока (работает как обычный Array.map).</p>

<p><code>filter(x => x > 10)</code> — пропускает дальше только те значения, которые удовлетворяют условию.</p>

<p><code>tap(x => console.log(x))</code> — позволяет выполнить побочное действие (например, логирование), не изменяя сами данные потока.</p>

<h5>3. Операторы высшего порядка (Flattening):</h5>

<p><code>switchMap</code> — отменяет предыдущий внутренний поток и переключается на новый. Идеален для HTTP-запросов (например, при поиске с автодополнением, чтобы отменить старый запрос, если пользователь ввел новую букву).</p>

<p><code>mergeMap</code> — выполняет все внутренние потоки параллельно, не соблюдая порядок.</p>

<p><code>concatMap</code> — ставит внутренние потоки в очередь и выполняет их строго друг за другом.</p>

<h5>4. Управление жизненным циклом:</h5>

<p><code>take(5)</code> — берет только первые 5 значений и автоматически завершает поток.</p>

<p><code>takeUntil(notifier$)</code> — слушает поток до тех пор, пока не сработает другой поток (notifier$). Незаменим для автоматической отписки при уничтожении компонента в Angular.</p>
