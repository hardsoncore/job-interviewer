<p>
  <span class="accent">Утилитарные операторы</span> помогают управлять потоками данных, выполняя различные вспомогательные операции.
  Они не изменяют сами значения, а предоставляют дополнительные возможности для работы с потоками.
</p>

<h3>tap</h3>
<p>
  Оператор <code>tap</code> позволяет выполнять побочные эффекты для каждого значения, проходящего через поток, не изменяя сами значения.
  <br>
  Это полезно для логирования, отслеживания или выполнения других действий без влияния на поток.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { tap } from 'rxjs/operators';

  of(1, 2, 3, 4, 5).pipe(
    tap(value => console.log('Текущее значение:', value))
  ).subscribe();
  // → Текущее значение: 1
  // → Текущее значение: 2
  // → Текущее значение: 3
  // → Текущее значение: 4
  // → Текущее значение: 5
</code>

<h3>finalize</h3>
<p>
  Оператор <code>finalize</code> позволяет выполнять побочные эффекты, когда поток завершается, независимо от того, произошло ли это успешно или с ошибкой.
  <br>
  Это полезно для очистки ресурсов, логирования завершения или выполнения других действий при завершении потока.
</p>

<code class="code">
  import { of } from 'rxjs';
  import { finalize } from 'rxjs/operators';

  of(1, 2, 3).pipe(
    finalize(() => console.log('Поток завершён'))
  ).subscribe();
  // → 1
  // → 2
  // → 3
  // → Поток завершён
</code>

<h3>delay</h3>
<p>
  Оператор <code>delay</code> позволяет задерживать эмиссию значений на определённое время.
  <br>
  Это полезно для создания искусственных задержек, управления временем эмиссии или синхронизации с другими потоками.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { delay } from 'rxjs/operators';

  of('Привет').pipe(
    delay(2000) // Задержка в 2 секунды
  ).subscribe(value => console.log(value));
  // → (через 2 секунды) Привет
</code>

<h3>timeout</h3>
<p>
  Оператор <code>timeout</code> позволяет установить максимальное время ожидания для эмиссии значения. Если значение не будет эмитировано в течение этого времени, поток завершится с ошибкой.
  <br>
  Это полезно для обработки ситуаций, когда ожидается ответ в течение определённого времени (например, при работе с сетевыми запросами) и для предотвращения зависания приложения.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { timeout } from 'rxjs/operators';

  of('Ответ').pipe(
    timeout(1000) // Установка таймаута в 1 секунду
  ).subscribe(
    value => console.log(value),
    error => console.log('Ошибка:', error)
  );
  // → Ответ (если эмитировано в течение 1 секунды)
  // → Ошибка: TimeoutError (если не эмитировано в течение 1 секунды)
</code>

<h3>retry</h3>
<p>
  Оператор <code>retry</code> позволяет повторять поток при возникновении ошибки определённое количество раз.
  <br>
  Это полезно для обработки временных ошибок, таких как сетевые запросы, и обеспечения устойчивости приложения.
</p>
<code class="code">
  import { of, throwError } from 'rxjs';
  import { retry } from 'rxjs/operators';

  of('Запрос').pipe(
    mergeMap(value => Math.random() < 0.5 ? throwError('Ошибка') : of(value)),
    retry(3) // Повторить до 3 раз при ошибке
  ).subscribe(
    value => console.log('Успех:', value),
    error => console.log('Ошибка после 3 попыток:', error)
  );
</code>
