<p>Процесс превращения HTML, CSS и JS кода в интерактивную страницу на экране называется <strong>критическим путем рендеринга (Critical Rendering Path)</strong>. Он состоит из следующих последовательных шагов:</p>

<h3>1. Парсинг HTML и построение DOM-дерева</h3>
<p>Как только браузер получает первые байты HTML-документа по сети, запускается HTML-парсер.</p>

<p>Парсер читает код сверху вниз, преобразуя теги в токены (открытие тега, контент, закрытие тега), а токены — в узлы (nodes).</p>

<p>Из этих узлов выстраивается <span class="accent">DOM (Document Object Model)</span> — древовидная структура, описывающая содержимое страницы.</p>

<p class="info">Важно: Парсинг HTML происходит инкрементально (по мере скачивания), но может быть заблокирован синхронными скриптами (<code>&lt;script&gt;</code>).</p>

<h3>2. Загрузка внешних ресурсов и выполнение JavaScript</h3>
<p>Встречая ссылки на внешние ресурсы (шрифты, картинки, CSS, JS), браузер отправляет запросы на их скачивание.</p>

<p>Особую роль играет JavaScript:</p>

<p> - По умолчанию <code>&lt;script&gt;</code> блокирует парсинг HTML. Браузер останавливает построение DOM, скачивает скрипт, выполняет его, и только потом продолжает парсить HTML.</p>

<p> - <code>defer</code> — скрипт скачивается в фоне, не блокируя парсинг, и выполняется строго после того, как весь HTML проанализирован, сохраняя порядок подключения скриптов.</p>

<p> - <code>async</code> — скрипт скачивается в фоне и выполняется сразу же после скачивания, приостанавливая парсинг HTML на время выполнения. Порядок выполнения не гарантируется.</p>

<p>Современные браузеры используют Preload Scanner — фоновый процесс, который "забегает" вперед парсера HTML, находит внешние ресурсы и начинает скачивать их заранее.</p>

<h3>3. Парсинг CSS и создание CSSOM</h3>
<p>Как только скачивается CSS-файл, браузер преобразует его в <span class="accent">CSSOM (CSS Object Model)</span> — древовидную структуру, описывающую стили для узлов DOM.</p>

<p>В отличие от HTML, парсинг CSS не может быть инкрементальным, так как стили, объявленные в конце файла, могут переопределять стили из начала.</p>

<p>CSS блокирует рендеринг (Render-Blocking). Браузер не начнет отрисовывать страницу, пока не будет построен полный CSSOM, чтобы избежать "мерцания нестилизованного контента" (FOUC).</p>

<h3>4. Построение дерева рендеринга (Render Tree)</h3>
<p>Браузер объединяет DOM и CSSOM в единое дерево рендеринга (Render Tree).</p>

<p>Render Tree содержит только те узлы, которые будут визуально отображены на экране.</p>

<p>Теги вроде <code>&lt;head&gt;</code>, <code>&lt;meta&gt;</code>, <code>&lt;script&gt;</code> не попадают в это дерево.</p>

<p>Узлы со свойством <code>display: none</code> исключаются из дерева рендеринга.</p>

<p>Узлы со стилями <code>visibility: hidden</code> или <code>opacity: 0</code> остаются в дереве, так как они продолжают занимать место на экране (влияют на геометрию).</p>

<h3>5. Вычисление геометрии (Layout / Reflow)</h3>
<p>Имея Render Tree, браузер знает, что рисовать, но не знает, где.</p>
<p>На этапе <span class="accent">Layout (Компоновка)</span> браузер проходит по дереву рендеринга сверху вниз и вычисляет точные размеры (ширину, высоту) и координаты каждого элемента на экране (в пикселях) относительно вьюпорта.</p>

<p><span class="accent">Reflow (Перекомпоновка)</span> — это процесс, при котором браузер заново вычисляет размеры и точное местоположение элементов на странице. По сути, это повторное выполнение шага «Расчет макета» (Layout).</p>

<h3>6. Отрисовка (Paint)</h3>
<p>Браузер знает геометрию всех элементов. На этапе <span class="accent">Paint</span> он начинает заполнять пиксели на экране: рисует текст, цвета, границы, тени и фоновые изображения. Отрисовка обычно происходит на нескольких независимых слоях (layers).</p>

<h3>7. Композитинг (Composite)</h3>
<p>Так как элементы могли быть отрисованы на разных слоях (например, элементы с <code>transform</code>, <code>opacity</code> или <code>z-index</code>), на последнем этапе браузер накладывает эти слои друг на друга в правильном порядке на графическом процессоре (GPU), формируя итоговое изображение, которое видит пользователь.</p>

<h3>Ключевые события загрузки</h3>
<p><code>DOMContentLoaded</code> — срабатывает, когда исходный HTML полностью загружен и распарсен в DOM, без ожидания загрузки таблиц стилей, изображений и фреймов.</p>

<p><code>load</code> — срабатывает, когда страница загружена полностью, включая все зависимые ресурсы (картинки, стили и т.д.).</p>
