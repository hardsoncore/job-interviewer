<h4>Шаг 1: Суть проблемы</h4>
<p>Представь, что тебе нужна функция, которая принимает аргумент и просто возвращает его обратно. Без дженериков у тебя два пути, и оба имеют критические недостатки:</p>

<p><strong>Жесткая типизация:</strong> Привязка функции к конкретному типу (например, string). Это делает код непереиспользуемым для чисел или объектов.</p>

<p><strong>Использование <code>any</code>:</strong> Ты получаешь универсальность, но полностью теряешь type safety. IDE перестает подсказывать методы, а компилятор пропустит ошибки во время сборки.</p>

<h4>Шаг 2: Решение — базовый синтаксис</h4>

<p><span class="accent">Дженерики (обобщения)</span> — это переменные для типов. Они позволяют определить тип динамически в момент вызова функции, создания класса или интерфейса.</p>

<code class="code">
  // &lt;T> — это placeholder (переменная типа), который "захватит" переданный тип
  function identity&lt;T>(arg: T): T {
    return arg;
  }

  // Теперь компилятор строго связывает входной и выходной типы:
  const myString = identity&lt;string>("Hello"); // Возвращает string
  const myNumber = identity&lt;number>(42);      // Возвращает number
</code>

<h4>Шаг 2.1: Покрытие входных и выходных типов</h4>

<p>Есть случаи, когда функция принимает один тип данных, а возвращает другой. В таких ситуациях ты можешь использовать несколько параметров типа.</p>

<p>Это позволяет тебе сохранять строгую типизацию для обоих типов, обеспечивая при этом гибкость в работе с различными данными.</p>

<code class="code">
  // Функция принимает тип входа (TInput) и тип выхода (TOutput)
  function processData&lt;TInput, TOutput>(
    input: TInput,
    transformer: (val: TInput) => TOutput
  ): TOutput {
    return transformer(input);
  }

  // Пример использования: преобразуем строку в число
  const result = processData&lt;string, number>("123", (val) => parseInt(val));
</code>

<h4>Шаг 3: Дженерики в интерфейсах (Реальный контракт)</h4>

<p>При работе с REST API структура ответа сервера часто стандартизирована, меняется только само тело полезной нагрузки (payload). Дженерики идеально описывают такие паттерны, исключая дублирование кода.</p>

<code class="code">
  // Базовый интерфейс, где T — это тип полезной нагрузки
  interface ServerResponse&lt;T> {
    status: number;
    message: string;
    data: T;
  }

  interface User { id: number; name: string; }
  interface Article { title: string; content: string; }

  // Переиспользование базового ответа для разных сущностей
  const userResponse: ServerResponse&lt;User> = {
    status: 200,
    message: "OK",
    data: { id: 1, name: "John" }
  };

  const articleResponse: ServerResponse&lt;Article> = { /* ... */ };
</code>

<h4>Шаг 4: Ограничения дженериков (Constraints)</h4>

<p>Иногда тебе нужно, чтобы тип T не был абсолютно любым, а гарантированно имел определенную структуру (например, метод или свойство). Для этого используется ключевое слово <code>extends</code>.</p>

<code class="code">
  interface HasLength {
    length: number;
  }

  // T может быть любым типом, при условии, что в нем есть свойство length
  function logLength&lt;T extends HasLength>(arg: T): T {
    console.log(arg.length);
    return arg;
  }

  logLength("Test");      // Ок (у строки есть length)
  logLength([1, 2, 3]);   // Ок (у массива есть length)
  // logLength(123);      // Ошибка компиляции: у числа нет свойства length
</code>

<h4>Шаг 5: Множественные параметры типа</h4>

<p>Ты можешь использовать несколько параметров типа, если функция или класс работает с несколькими типами данных.</p>
<code class="code">
  function merge&lt;T, U>(obj1: T, obj2: U): T & U {
    return { ...obj1, ...obj2 };
  }

  const merged = merge({ name: "Alice" }, { age: 30 });
  // merged имеет тип { name: string; age: number }
</code>


<h4>Шаг 6: Дженерики в классах</h4>
<p>Дженерики не ограничиваются функциями. Ты можешь создавать классы, которые работают с любыми типами данных, сохраняя при этом строгую типизацию.</p>
<code class="code">
  class Box&lt;T> {
    private content: T;

    constructor(content: T) {
      this.content = content;
    }

    getContent(): T {
      return this.content;
    }

    setContent(content: T): void {
      this.content = content;
    }
  }
</code>

<h4>Дженерики в Utility Types</h4>
<p>TypeScript поставляется с набором встроенных утилит, которые используют дженерики для создания новых типов на основе существующих. Например, <code>Partial&lt;T></code> делает все свойства типа <code>T</code> необязательными, а <code>Readonly&lt;T></code> делает все свойства типа <code>T</code> только для чтения.</p>

<p><code>Pick&lt;T, K></code> позволяет выбрать подмножество свойств из типа T, а <code>Record&lt;K, T></code> создает тип объекта с ключами K и значениями T. Эти утилиты широко используются в реальных проектах для управления типами и обеспечения безопасности типов при работе с объектами и функциями.</p>

<h4>Шаг 7: Дженерики в TypeScript 4.7 и выше</h4>
<p>С выходом TypeScript 4.7 появились новые возможности для дженериков, такие как улучшенная поддержка для рекурсивных типов и более гибкие ограничения. Теперь ты можешь создавать сложные типы, которые раньше были невозможны или требовали громоздких решений.</p>
<p>Например, рекурсивные типы позволяют описывать структуры данных, которые могут содержать вложенные элементы того же типа, что особенно полезно при работе с деревьями или графами. Улучшенные ограничения позволяют создавать более точные типы, которые лучше отражают реальные данные и бизнес-логику.</p>

<h4>Как Generics работают под капотом в JavaScript</h4>
<p>Важно понимать, что дженерики — это исключительно инструмент для статической типизации в TypeScript. Когда код компилируется в JavaScript, все типы, включая дженерики, удаляются. Это означает, что в рантайме нет информации о типах, и JavaScript не знает о существовании дженериков. Они служат только для обеспечения безопасности типов во время разработки и компиляции, но не влияют на поведение кода в рантайме.</p>

<h4>Заключение</h4>

<p><span class="accent">Дженерики</span> — это мощный инструмент для создания гибкого и при этом строго типизированного кода. Они позволяют тебе писать функции и классы, которые работают с любыми типами данных, не жертвуя безопасностью типов. В современном TypeScript ты будешь использовать дженерики на каждом шагу, от простых утилит до сложных архитектурных паттернов.</p>
