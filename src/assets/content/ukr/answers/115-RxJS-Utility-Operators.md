<p>
  <span class="accent">Утилітарні оператори</span> допомагають керувати потоками даних, виконуючи різноманітні допоміжні операції.
  Вони не змінюють самі значення, а надають додаткові можливості для роботи з потоками.
</p>

<h3>tap</h3>
<p>
  Оператор <code>tap</code> дозволяє виконувати побічні ефекти для кожного значення, що проходить через потік, не змінюючи самі значення.
  <br>
  Це корисно для логування, відстеження або виконання інших дій без впливу на потік.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { tap } from 'rxjs/operators';

  of(1, 2, 3, 4, 5).pipe(
    tap(value => console.log('Поточне значення:', value))
  ).subscribe();
  // → Поточне значення: 1
  // → Поточне значення: 2
  // → Поточне значення: 3
  // → Поточне значення: 4
  // → Поточне значення: 5
</code>

<h3>finalize</h3>
<p>
  Оператор <code>finalize</code> дозволяє виконувати побічні ефекти, коли потік завершується, незалежно від того, сталося це успішно чи з помилкою.
  <br>
  Це корисно для очищення ресурсів, логування завершення або виконання інших дій під час завершення потоку.
</p>

<code class="code">
  import { of } from 'rxjs';
  import { finalize } from 'rxjs/operators';

  of(1, 2, 3).pipe(
    finalize(() => console.log('Потік завершено'))
  ).subscribe();
  // → 1
  // → 2
  // → 3
  // → Потік завершено
</code>

<h3>delay</h3>
<p>
  Оператор <code>delay</code> дозволяє затримувати емісію значень на певний час.
  <br>
  Це корисно для створення штучних затримок, керування часом емісії або синхронізації з іншими потоками.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { delay } from 'rxjs/operators';

  of('Привіт').pipe(
    delay(2000) // Затримка у 2 секунди
  ).subscribe(value => console.log(value));
  // → (через 2 секунди) Привіт
</code>

<h3>timeout</h3>
<p>
  Оператор <code>timeout</code> дозволяє встановити максимальний час очікування для емісії значення. Якщо значення не буде емітовано протягом цього часу, потік завершиться з помилкою.
  <br>
  Це корисно для обробки ситуацій, коли очікується відповідь протягом певного часу (наприклад, під час роботи з мережевими запитами) та для запобігання зависанню застосунку.
</p>
<code class="code">
  import { of } from 'rxjs';
  import { timeout } from 'rxjs/operators';

  of('Відповідь').pipe(
    timeout(1000) // Встановлення таймаута в 1 секунду
  ).subscribe(
    value => console.log(value),
    error => console.log('Помилка:', error)
  );
  // → Відповідь (якщо емітовано протягом 1 секунди)
  // → Помилка: TimeoutError (якщо не емітовано протягом 1 секунди)
</code>

<h3>retry</h3>
<p>
  Оператор <code>retry</code> дозволяє повторювати потік у разі виникнення помилки певну кількість разів.
  <br>
  Це корисно для обробки тимчасових помилок, таких як мережеві запити, та забезпечення стійкості застосунку.
</p>
<code class="code">
  import { of, throwError } from 'rxjs';
  import { retry, mergeMap } from 'rxjs/operators';

  of('Запит').pipe(
    mergeMap(value => Math.random() < 0.5 ? throwError('Помилка') : of(value)),
    retry(3) // Повторити до 3 разів у разі помилки
  ).subscribe(
    value => console.log('Успіх:', value),
    error => console.log('Помилка після 3 спроб:', error)
  );
</code>
