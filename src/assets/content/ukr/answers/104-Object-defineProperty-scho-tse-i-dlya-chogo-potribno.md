<p>
  <code>Object.defineProperty()</code> — це статичний метод, який дозволяє безпосередньо визначити нову властивість на об'єкті або змінити існуючу, а потім повертає цей об'єкт.
</p>

<p>
  Цей метод корисний, коли вам потрібно створити властивість з певними характеристиками, такими як незмінність, неперераховуваність або неконфігурованість. Він також дозволяє створювати гетери та сетери для управління доступом до властивості.

  Щодо практичного застосування, <code>Object.defineProperty()</code> часто використовується в бібліотеках та фреймворках для створення API з певними правилами доступу до даних, а також для оптимізації продуктивності шляхом контролю над тим, як властивості взаємодіють з механізмами JavaScript.
</p>

<h4>Синтаксис</h4>

<code class="code">
  Object.defineProperty(obj, prop, descriptor);
</code>

<p>
  <code>obj</code> — об'єкт, на якому буде визначена властивість.
</p>
<p>
  <code>prop</code> — ім'я властивості, яка буде визначена або змінена.
</p>
<p>
  <code>descriptor</code> — об'єкт, який описує конфігурацію властивості. Він може містити такі ключі:
</p>

<p>
  Дескриптори бувають двох типів: <strong>дескриптори даних</strong> та <strong>дескриптори доступу</strong>. Залежно від типу дескриптора, набір допустимих ключів буде відрізнятися.
</p>

<p>Спільні ключі дескриптора (common descriptor keys) - доступні для обох типів</p>
<ul>
  <li><code>enumerable</code> — вказує, чи буде властивість перераховуватися під час перебору об'єкта. За замовчуванням <code>false</code>.</li>
  <li><code>configurable</code> — вказує, чи може властивість бути видалена або змінена. За замовчуванням <code>false</code>.</li>
</ul>

<p>Дескриптори даних (data descriptors) - описують класичне значення властивості</p>
<ul>
  <li><code>value</code> — значення властивості. За замовчуванням <code>undefined</code>.</li>
  <li><code>writable</code> — вказує, чи може значення властивості бути змінено. За замовчуванням <code>false</code>.</li>
</ul>

<p>Дескриптори доступу (accessor descriptors) - описують властивості з гетерами та сетерами</p>
<ul>
  <li><code>get</code> — (за замовчуванням <code>undefined</code>): Функція, яка неявно викликається без аргументів при будь-якому зверненні до властивості. Значення, що повертається цією функцією, стає значенням властивості.</li>
  <li><code>set</code> — (за замовчуванням <code>undefined</code>): Функція, яка викликається з одним аргументом (новим значенням) при спробі присвоїти властивості нове значення.</li>
</ul>

<p class="info info--orange">
  Важливе правило: дескриптор не може бути одночасно і тим, і іншим. Якщо вказати <code>value</code> та <code>get</code> в одному виклику, JavaScript викине помилку:

  <code class="code">
    Object.defineProperty(obj, 'prop', {
    value: 42,
    get() { return this._prop; }
    });

    // TypeError: Invalid property descriptor. Cannot both specify accessors and a value or writable attribute
  </code>

</p>

<h4>Приклад використання</h4>
<code class="code">
  const obj = {};

  // Визначаємо властивість 'name' за допомогою Object.defineProperty
  Object.defineProperty(obj, 'name', {
  value: 'Alice',
  writable: false, // Властивість 'name' не можна змінити
  enumerable: true,
  configurable: false // Властивість 'name' не можна видалити або змінити її конфігурацію
  });

  console.log(obj.name); // Alice

  // Намагаємося змінити значення властивості 'name'
  obj.name = 'Bob';
  console.log(obj.name); // Alice

  // Робимо спробу видалити властивість 'name'
  delete obj.name;
  console.log(obj.name); // Alice
</code>

<h4>Навіщо це використовувати в повсякденній практиці?</h4>
<p>
  У звичайних продуктових завданнях рідко доводиться використовувати <code>Object.defineProperty</code> напряму. Для гетерів та сетерів зараз частіше використовують більш лаконічний синтаксис класів або літералів об'єктів (наприклад, <code>get propName() { ... }</code>).

  Однак розуміння того, як працює <code>Object.defineProperty</code>, є важливим для глибокого розуміння JavaScript, особливо якщо ти плануєш працювати з фреймворками, які активно використовують цю функцію для створення реактивних властивостей або оптимізації продуктивності.

</p>

<h4>Використання у Vue 2</h4>

<p>
  У Vue <code>Object.defineProperty</code> використовується для створення реактивних властивостей, відстеження змін у даних та автоматичного оновлення DOM при зміні цих даних.
</p>

<p>
  <code>Object.defineProperty</code> як фундаментальна основа реактивності — це ексклюзивна риса Vue 2. Інші фреймворки (навіть наступні покоління самого Vue) пішли зовсім іншими шляхами.
</p>
