<h4>Крок 1: Суть проблеми</h4>
<p>Уяви, що тобі потрібна функція, яка приймає аргумент і просто повертає його назад. Без дженериків у тебе два шляхи, і обидва мають критичні недоліки:</p>

<p><strong>Жорстка типізація:</strong> Прив'язка функції до конкретного типу (наприклад, string). Це робить код неперевикористовуваним для чисел або об'єктів.</p>

<p><strong>Використання <code>any</code>:</strong> Ти отримуєш універсальність, але повністю втрачаєш type safety. IDE перестає підказувати методи, а компілятор пропустить помилки під час збірки.</p>

<h4>Крок 2: Рішення — базовий синтаксис</h4>

<p><span class="accent">Дженерики (узагальнення)</span> — це змінні для типів. Вони дозволяють визначити тип динамічно в момент виклику функції, створення класу або інтерфейсу.</p>

<code class="code">
  // &lt;T> — це placeholder (змінна типу), який "захопить" переданий тип
  function identity&lt;T>(arg: T): T {
    return arg;
  }

  // Тепер компілятор суворо пов'язує вхідний та вихідний типи:
  const myString = identity&lt;string>("Hello"); // Повертає string
  const myNumber = identity&lt;number>(42);      // Повертає number
</code>

<h4>Крок 2.1: Покриття вхідних та вихідних типів</h4>

<p>Є випадки, коли функція приймає один тип даних, а повертає інший. У таких ситуаціях ти можеш використовувати кілька параметрів типу.</p>

<p>Це дозволяє тобі зберігати сувору типізацію для обох типів, забезпечуючи при цьому гнучкість у роботі з різними даними.</p>

<code class="code">
  // Функція приймає тип входу (TInput) і тип виходу (TOutput)
  function processData<TInput, TOutput>(
    input: TInput,
    transformer: (val: TInput) => TOutput
  ): TOutput {
    return transformer(input);
  }

  // Приклад використання: перетворюємо рядок на число
  const result = processData<string, number>("123", (val) => parseInt(val));
</code>

<h4>Крок 3: Дженерики в інтерфейсах (Реальний контракт)</h4>

<p>При роботі з REST API структура відповіді сервера часто стандартизована, змінюється лише саме тіло корисного навантаження (payload). Дженерики ідеально описують такі патерни, виключаючи дублювання коду.</p>

<code class="code">
  // Базовий інтерфейс, де T — це тип корисного навантаження
  interface ServerResponse&lt;T> {
    status: number;
    message: string;
    data: T;
  }

  interface User { id: number; name: string; }
  interface Article { title: string; content: string; }

  // Перевикористання базової відповіді для різних сутностей
  const userResponse: ServerResponse&lt;User> = {
    status: 200,
    message: "OK",
    data: { id: 1, name: "John" }
  };

  const articleResponse: ServerResponse&lt;Article> = { /* ... */ };
</code>

<h4>Крок 4: Обмеження дженериків (Constraints)</h4>

<p>Іноді тобі потрібно, щоб тип T не був абсолютно будь-яким, а гарантовано мав певну структуру (наприклад, метод або властивість). Для цього використовується ключове слово <code>extends</code>.</p>

<code class="code">
  interface HasLength {
    length: number;
  }

  // T може бути будь-яким типом, за умови, що в ньому є властивість length
  function logLength&lt;T extends HasLength>(arg: T): T {
    console.log(arg.length);
    return arg;
  }

  logLength("Test");      // Ок (у рядка є length)
  logLength([1, 2, 3]);   // Ок (у масиву є length)
  // logLength(123);      // Помилка компіляції: у числа немає властивості length
</code>

<h4>Крок 5: Множинні параметри типу</h4>

<p>Ти можеш використовувати кілька параметрів типу, якщо функція або клас працює з кількома типами даних.</p>
<code class="code">
  function merge&lt;T, U>(obj1: T, obj2: U): T & U {
    return { ...obj1, ...obj2 };
  }

  const merged = merge({ name: "Alice" }, { age: 30 });
  // merged має тип { name: string; age: number }
</code>


<h4>Крок 6: Дженерики в класах</h4>
<p>Дженерики не обмежуються функціями. Ти можеш створювати класи, які працюють з будь-якими типами даних, зберігаючи при цьому сувору типізацію.</p>
<code class="code">
  class Box&lt;T> {
    private content: T;

    constructor(content: T) {
      this.content = content;
    }

    getContent(): T {
      return this.content;
    }

    setContent(content: T): void {
      this.content = content;
    }
  }
</code>

<h4>Дженерики в Utility Types</h4>
<p>TypeScript постачається з набором вбудованих утиліт, які використовують дженерики для створення нових типів на основі існуючих. Наприклад, <code>Partial&lt;T&gt;</code> робить усі властивості типу <code>T</code> необов'язковими, а <code>Readonly&lt;T&gt;</code> робить усі властивості типу <code>T</code> лише для читання.</p>

<p><code>Pick&lt;T, K&gt;</code> дозволяє вибрати підмножину властивостей з типу <code>T</code>, а <code>Record&lt;K, T&gt;</code> створює тип об'єкта з ключами <code>K</code> і значеннями <code>T</code>. Ці утиліти широко використовуються в реальних проєктах для керування типами та забезпечення безпеки типів при роботі з об'єктами та функціями.</p>

<h4>Крок 7: Дженерики в TypeScript 4.7 і вище</h4>
<p>З виходом TypeScript 4.7 з'явилися нові можливості для дженериків, такі як покращена підтримка для рекурсивних типів і більш гнучкі обмеження. Тепер ти можеш створювати складні типи, які раніше були неможливі або вимагали громіздких рішень.</p>
<p>Наприклад, рекурсивні типи дозволяють описувати структури даних, які можуть містити вкладені елементи того ж типу, що особливо корисно при роботі з деревами або графами. Покращені обмеження дозволяють створювати більш точні типи, які краще відображають реальні дані та бізнес-логіку.</p>

<h4>Як Generics працюють під капотом у JavaScript</h4>
<p>Важливо розуміти, що дженерики — це виключно інструмент для статичної типізації в TypeScript. Коли код компілюється в JavaScript, всі типи, включаючи дженерики, видаляються. Це означає, що під час виконання немає інформації про типи, і JavaScript не знає про існування дженериків. Вони служать лише для забезпечення безпеки типів під час розробки та компіляції, але не впливають на поведінку коду під час виконання.</p>

<h4>Висновок</h4>

<p><span class="accent">Дженерики</span> — це потужний інструмент для створення гнучкого і при цьому суворо типізованого коду. Вони дозволяють тобі писати функції та класи, які працюють з будь-якими типами даних, не жертвуючи безпекою типів. У сучасному TypeScript ти будеш використовувати дженерики на кожному кроці, від простих утиліт до складних архітектурних патернів.</p>
