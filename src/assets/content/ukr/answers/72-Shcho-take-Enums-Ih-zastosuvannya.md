<h3>Що таке Enums?</h3>

<p><span class="accent">Enum</span> (скорочення від enumeration — перерахування) — це структура даних, яка дозволяє задати суворий набір іменованих констант, логічно пов'язаних між собою.</p>

<p>Грубо кажучи, це словник дозволених станів або типів для конкретної сутності. Замість того, щоб використовувати в коді «магічні числа» (<code>1</code>, <code>2</code>, <code>3</code>) або звичайні рядки (<code>"active"</code>, <code>"deleted"</code>), ти створюєш єдине джерело істини.</p>

<h3>Навіщо їх застосовувати (і який біль вони вирішують)?</h3>

<h4>1. Код, що самодокументується</h4>
<p>Конструкція <code>if (user.role === UserRole.Admin)</code> читається як звичайний англійський текст. Це в рази зрозуміліше, ніж <code>if (user.role === 1)</code>.</p>

<h4>2. Захист від дурних помилок</h4>
<p>Якщо ти зробиш одруківку в слові (наприклад, напишеш <code>UserRole.Adimn</code>), компілятор повідомить про це ще до запуску коду. Зі звичайними рядками помилка виникла б тільки в рантаймі.</p>

<h4>3. Розумне автодоповнення</h4>
<p>IDE миттєво підкаже всі доступні варіанти, щойно ти надрукуєш ім'я перерахування і поставиш крапку. Не потрібно тримати в голові, які там статуси є на бекенді.</p>

<h4>4. Централізований рефакторинг</h4>
<p>Якщо бекенд вирішив змінити значення статусу з <code>in_progress</code> на <code>processing</code>, ти змінюєш це значення рівно в одному рядку — всередині самого Enum.</p>

<h3>Види Enums</h3>
<p>У TypeScript є кілька видів Enums:</p>

<p><strong>Numeric Enums</strong> — за замовчуванням кожен елемент отримує числове значення, починаючи з 0. Наприклад:</p>
<code class="code">
  enum UserRole {
    Admin, // 0
    User,  // 1
    Guest  // 2
  }

  // Ти можеш явно задавати значення, якщо хочеш:
  enum ResponseStatus {
    Pending = 0,
    Success = 200,
    Failed = 404
  }
</code>

<p><strong>String Enums</strong> — кожен елемент явно задається рядком. Це більш читабельний і безпечний варіант:</p>
<code class="code">
  enum UserRole {
    Admin = "ADMIN",
    User = "USER",
    Guest = "GUEST"
  }
</code>

<p><strong>Heterogeneous Enums</strong> — змішані, де елементи можуть бути як рядками, так і числами. Зазвичай їх не рекомендують використовувати, оскільки це може заплутати.</p>

<p class="info info--orange">Вважаються антипатерном, оскільки можуть призвести до неочевидних помилок.</p>

<code class="code">
  enum MixedEnum {
    No = 0,
    Yes = "YES"
  }
</code>

<p><strong>Const Enums</strong> — оптимізовані для продуктивності, вони повністю видаляються під час компіляції, а їхні значення вставляються безпосередньо в код. Це може бути корисно для констант, що часто використовуються, але варто використовувати з обережністю, оскільки вони не підтримують деякі можливості звичайних Enums (наприклад, ітерацію). Також трапляються проблеми з налагодженням, оскільки у скомпільованому коді не буде явного об'єкта Enum. Останнім часом їх використання стало менш популярним.</p>

<code class="code">
  const enum UserRole {
    Admin = "ADMIN",
    User = "USER",
    Guest = "GUEST"
  }
</code>

<h3>Що "під капотом" і як бути в чистому JS?</h3>
<p>Важливо пам'ятати, що в нативному JavaScript вбудованих Enums не існує.</p>
<h4>Що відбувається під час компіляції?</h4>

<p>Коли TypeScript компілює код, він перетворює твої Enums на реальні JavaScript-об'єкти (створюючи так звані IIFE-функції).</p>
<p>Наприклад, Enum <code>UserRole</code> буде скомпільований в об'єкт, який можна використовувати для доступу до його значень. Це дозволяє зберегти переваги Enums навіть у середовищі, де їх спочатку немає.</p>

<code class="code">
  // TypeScript Enum
  enum Role {
    Admin = 0,
    User = 1
  }

  // Скомпільований JavaScript
  "use strict";
  var Role;
  (function (Role) {
    Role[Role["Admin"] = 0] = "Admin";
    Role[Role["User"] = 1] = "User";
  })(Role || (Role = {}));
</code>

<h4>Як бути в чистому JS?</h4>

<p>Якщо ти працюєш над проєктом без TypeScript (наприклад, у класичному Vue 2 на чистому JS), паттерн Enum легко емулюється через незмінний об'єкт:</p>

<code class="code">
  const TaskStatus = Object.freeze({
    TODO: 'TODO',
    IN_PROGRESS: 'IN_PROGRESS',
    DONE: 'DONE'
  });

  // Використання точно таке саме: TaskStatus.DONE
</code>
